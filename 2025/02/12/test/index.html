extern static inline

extern

当使用 extern 关键字声明一个函数或变量时，该函数或变量并不会被定义，而是告诉编译器这个变量在其他翻译单元已经定义好了。例如，我们可以在一个源文件中定义一个全局变量，然后在其他的源文件中使用该变量，而不必重新定义一遍。简单来说就是，extern 可以告诉编译器某个变量已经在其他源文件中定义过了。

static

不同翻译单元可以有同名的static变量或函数，但该变量或函数只对对应的翻译单元可见。如果同一定义的 static变量或函数，被不同翻译单元调用，每个编译单元有自己单独的一份拷贝，且此拷贝只对 对应的翻译单元可见。

inline

inline 说明符 - cppreference.com

现代的编译器在决定是否将函数调用进行内联展开时，几乎不参考函数声明中inline修饰符

带外部链接的内联函数或变量 (C++17 起)（例如不声明为 static）拥有下列额外属性：

  • 内联函数或变量 (C++17 起)在程序中可以有多于一次定义，只要每个定义都出现在不同翻译单元中（对于非静态的内联函数和变量 (C++17 起)）且所有定义等同即可。例如，内联函数或内联变量 (C++17 起)可以在被多个源文件包含的头文件中定义
  • 它必须在每个翻译单元中都被声明为 inline
  • 它在每个翻译单元中都拥有相同的地址
